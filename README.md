## 👀 프로젝트 빌드 및 실행 방법

1. 상단 `Code` 버튼을 눌러 레포지토리를 클론 받습니다.

```
$ git clone https://github.com/wanted-team2/3week_SearchFilter.git

```

```

3. 서버를 실행합니다.

```

### 🏗 로직에 대한 설명 및 설계

#### 1. 검색 기능 debounce 적용
- 콜백 함수의 불필요한 실행을 줄여주기위해 검색 기능에 `debounce를` 적용 했다.<br> 불필요한 서버 리퀘스트를 막고, 불필요한 통신을 줄임과 동시에 필요 없는 렌더링 또한 막을 수 있어 컴포넌트의 성능 개선에도 도움을 준다.

#### 2. UI/UX
- 데스크탑과 모바일 두 가지 버전의 레이아웃을 구성하였다.
- 데스크탑 버전에서는 `페이지네이션`을 이용해 많은 양의 데이터를 조회가능하도록, 모바일 버전에서는 `브랜드 태그버튼`을 추가해 브랜드별 상품들을 쉽게 조회할 수 있도록 구성하였다.

#### 3. 검색어 입력 후 버튼 클릭 / 엔터키 입력 시 검색리스트 노출
- 사용자의 편의성을 높이기 위해 버튼 클릭뿐 아니라 키보드의 엔터키 입력 시에도 검색된 리스트가 보여지도록 설계했다.
